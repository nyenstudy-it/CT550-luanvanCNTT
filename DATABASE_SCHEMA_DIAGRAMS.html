Physical Data Model (PDM) & Conceptual Data Model (CDM)

💡 Mô hình Dữ liệu Khái niệm (CDM)

Sơ đồ này thể hiện các thực thể chính và mối quan hệ giữa chúng ở cấp độ khái niệm (không quan tâm đến chi tiết kỹ thuật).

📌 Giải thích CDM:
  • USER: Người dùng hệ thống (admin, staff, customer)
  • CUSTOMER: Khách hàng mua sắm
  • PRODUCT: Sản phẩm bán hàng
  • ORDER: Đơn hàng của khách
  • SUPPLIER: Nhà cung cấp sản phẩm
  • STAFF: Nhân viên công ty
  • PAYMENT: Thanh toán đơn hàng

📋 Danh sách Chi tiết Bảng

👥 Bảng Người Dùng & Xác thực

users
🔑 id - bigint, PK, AI
name - varchar(255)
email - varchar(255), UNI
password - varchar(255)
role - enum(admin, staff, customer)
status - enum(active, locked)
customers
🔑 id - bigint, PK, AI
🔗 user_id - bigint, FK, UNI
phone - varchar(255)
address - text
province, district, ward
gender - enum
staffs
🔑 user_id - bigint, PK, FK
phone - varchar(255)
position - enum(cashier, warehouse, order_staff)
employment_status
probation_hourly_wage
official_hourly_wage
sessions
🔑 id - varchar(255), PK
🔗 user_id - bigint, FK
ip_address - varchar(45)
user_agent - text
last_activity - int

description - text

status - enum(active, inactive)

🔗 product_id - bigint, FK

🔗 product_variant_id - bigint, FK, UNI
quantity - int
